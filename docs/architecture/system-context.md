# System Context

Настоящий документ кратко описывает будущие доменные контуры (bounded
contexts) EPD² Civic OS, как они определены в
`docs/canonical/TZ-00-domain-event-canon.md` (раздел 5).

На этапе CLAUDE-PACK-01 ни один из перечисленных контуров **не
реализован**. Ниже приводится только их назначение — без описания
реализации, технологий или внутренней структуры сервисов.

## Identity

Подтверждение личности участника: аккаунт, вход, MFA, проверка
уникальности, восстановление доступа. Не хранит политические действия
пользователя.

## Eligibility

Отвечает на вопрос, имеет ли подтверждённый участник право участвовать в
конкретном процессе (членство, регион, возрастные и временные условия).

## Credential

Технический мост между Identity и Participation: выдача ограниченных,
минимально связуемых Participation Credential.

## Organization

Организационная структура: организация, подразделения, Civic Spaces,
рабочие группы, роли, членство.

## Initiative

Инициативы, их версии, публикация, поддержка, поправки, источники,
прохождение этапов Programmwerkstatt.

## Deliberation

Обсуждения, аргументы, комментарии, ответы, реакции, структурированные
позиции.

## Moderation

Жалобы, флаги, модерационные решения, временное скрытие, ограничения,
апелляции, восстановление, публичная статистика.

## Voting

Параметры голосования, бюллетени, приём голосов, защита от повторного
участия, receipts, закрытие процесса. Не получает реальное имя участника.

## Tally

Проверка набора бюллетеней, подсчёт, повторяемость, обработка недействительных
записей, формирование подписанного результата.

## Delegation

Создание делегирования, scope, срок, отзыв, цепочки, циклы, snapshot, расчёт
итогового веса.

## Transparency

Публичный реестр инициатив и решений, история версий, результаты, журналы
модерации и ИИ, финансовая прозрачность, lobbying log, audit exports.

## Governance

Системные роли, политика полномочий, версии правил, emergency procedures,
crisis override, audit access, review procedures.
